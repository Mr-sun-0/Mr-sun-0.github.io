非Dom属性

  • ref
  1. 类似于Id可以定位真实dom,不能在函数组件上使用，因为它没有实例
  2. 但是可以在函数内部使用，定位到真实dom
  • key
  1. 提高渲染性能，唯一标识，index(不推荐)

Props介绍与应用

什么是props

  • 组件中的数据，一个是props，一个是state
  • 数据改变将会导致组件的渲染
    组件设置默认值
    组件.defaultProps =

react数据流是单向的，主要还是要保证组件的复用性

this的四种绑定方式 [^1]
  1. constructor bind 推荐
  2. jsx中bind绑定 会造成渲染问题，每次渲染都会渲染新的函数
  3. 箭头函数 推荐
  4. jsx中使用箭头函数

state介绍与应用

  • 构造函数是唯一可以给 this.state 赋值的地方,不要直接修改state,那将不会引起组件渲染
  • setState的更新可能是异步的，react会将state的多次更新合成一次调用
  • 如果需要依赖上一次的状态可以使用setSate((preState)=>{})

可控组件和不可控组件

受控组件
依赖状态，默认值实时映射到状态 state
必须使用onChange
双向绑定
非受控组件
只能操作dom节点

React进阶

ReactHook快速入门

  1. 为什么要用hooks?
    代码更加简洁
  • 声明周期难以理解，很难熟练运用
  • redux状态管理，概念非常多，难以理解，中文文档
  • 高阶组件理解起来不容易，必须掌握，似懂非懂
  • 优秀的解决方案，都在react社区，阅读困难
    hook学习成本降低
  • redux也不再是必需品，mobox上手非常容易
    开发成本降低，可以让组件维护自己的状态

  1. useState
    每次渲染，函数都会重新执行，函数执行完毕后，所有内存都会释放掉

  2. useEffect
    页面渲染后执行，第二个参数是定向依赖state变化，第一个函数可以返回一个操作在组件销毁时执行

  3. useContext
    需要重新学习，useContext会还原context的value值

  4. useReducer
    useState的替代方案，比如state变化依赖上一个状态
    接收三个参数

    import React,  { useReducer } from 'react';

    // 定义简单的reducer，第一个参数
    const reducer = (state, action) => {
        switch(action.type) {
            case 'add':
                return {...state, count: state.count + 1 };
            case 'reduce':
                return {...state, count: state.count - 1 }
            default:
                return state;
        }
    }

    // 定义第二个参数， createStorage的默认值
    let initialState = { count : 10, name : 'reducer' }

    // 定义第三个参数，一个函数, 会把第二个参数当参数执行
    const init = initialState => {
        return { count : initialState.count + 2}
    }

    export default function ReducerComponent() {
        const [state, dispatch] = useReducer(reducer, initialState, init);
        return (
            <div>
                <p>{state.count}</p>
                <button onClick = {()=> dispatch({type: 'add'})}>加</button>
                <button onClick = {()=> dispatch({type: 'reduce'})}>减</button>
            </div>
        )
    }

  5. useRef
    相当于绑定一个id，const ref = useRef(null),然后通过ref属性绑定到标签上

  6. useMemo和useCallbck
    useMemo 把创建函数和依赖项数组作为参数
    useCallback 接收一个内联回调函数和依赖项数组

React Router介绍

v2 v3 v4 v5
v4/v5用法和理念基本一致 v2 v3 差异较大

  1. 版本差异
    v4和v4之前版本比较：
    v4:是稳定版本，大多数项目都在，v4属于动态路由，之前是静态路由。react-router被拆分成多个包发布

  2. react-router

  • react-router
  • react-router-dom
  • react-router-native
  • react-router-config 静态路由配置助手
  1. v5和v4版本比较：
  • v5
    ○ >15的版本的完全兼容，更好支持支持react16
    ○ 完全兼容v4
    ○ 消除严格模式的警告
    ○ 适用creat-react-context 实现context API
    ○ 更注重稳定性和兼容性，无破坏性更新
    ○ 改进与新特性
    ○ 目前的React Router中，不需要路由配置，一切皆组件
前端路由
  1. 原理：检测url变化，截获url地址，然后进行url匹配
  2. 两种模式：

  • hash: 锚点

    • 刷新页面不会发送请求
    • demo
      <body>
          <button id="btn">点击</button>
          <script>
              const btn = document.getElementById("btn");
              btn.addEventListener("click", () => {
                  location.href = `#${Math.random() * 100}`;
              })
              window.addEventListener("hashchange", e => {
                  const { oldUrl, newUrl } = e;
                  console.log(oldUrl, newUrl, location.hash)
              });
          </script>
      </body>

  • html:history

    • pushState()
    • replaceState()
    • onpopstate事件
    • 刷新页面时会发生请求
React Router常见概念
  1. Router组件
  • 每个router都会创建一个history对象，用来保持当前位置的追踪
  • web 两种方式都会创建一个history对象
    • HashRouter 只处理相同的url
    • BrowserRouter 非静态的站点，要处理不同的url
  • react native
    • MemoryHistory
  1. Route组件
  • 只是一个具有渲染方法的react组件，路由匹配成功渲染对应组件
  • 常用属性
    • path:’’ 匹配规则 可以省略(没有path则默认匹配) 字符串类型
    • exact:boolean true时表示完全匹配
    • component 要渲染的组件
  • 不常用属性
    • render:函数形式 可以进行逻辑操作 path匹配的时候执行
    • children:函数形式 可以进行逻辑操作 不管匹不匹配都执行，参数中有match对象
  1. Switch组件
  • 最多只会匹配一个组件
  • 作用：可以将route组件分组
  1. Link与NavLink组件
  • 声明式的可访问导航
  • to:string | {pathname:’’,search:’’}
  • replace: boolean 替换当前的历史记录
  • NavLink - 特殊版的Link，当匹配的时候可以添加样式 - activeClassName activeStyle
  1. Redirect组件
  • to:string | object 必须的
  • push: boolean 会将新地址推到history中
  • from 将要重定向的url
  • exact:true
  1. History对象
  • 编程式导航，push方法
  1. withRouter组件
  • 没有经过路由匹配的组件，是取不到history对象的，就可以用withRouter包裹
  • 要注意withRouter的层级要小于BrowserRouter
  1. 动态路由
  • 路由规则不是预先决定的，而是渲染过程中决定的
  • 通过：来定义 => /:id
  • 通过 props.match.params.id来获取

踩坑记录

1. 为什么react事件要绑定this?

  • jsx语法实际是createElement的语法糖
    <div>Hello, { this.props.name }</div>
    // 等价于
    React.createElement( ‘div’,null, `Hello,${this.props.name}` )
  • createElement伪代码
    function createElement(dom, params) {
      var domObj = document.createElement(dom);
      domObj.onclick = params.onclick;
      domObj.innerHTML = params.conent;
      return domObj
    }
    可以看到，我们的自定义组件类中onclick绑定的事件，是作为回调函数绑定到domObj.onclick上的,button被点击时，会由React作为中介调用回调函数，此时的this指向丢失，就指向了window

要点

  1. hooks里的setState操作不是批处理的
  2. ref拿到的数据是实时的

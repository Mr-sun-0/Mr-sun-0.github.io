TypeScript学习记录

2021-12-16 万象 1.2k 字

为什么要使用TypeScript？

  1. 在v8编译过程中能更快的将字节码转成机器码，提升性能 ^1
  2. 代码规范 SOLID,静态检查可以避免一些不必要的bug，代码的可读性和可维护性都有显著的提高

如何用好TypeScript

  1. 两个文件一定要好好看
    a. lib.dom.d.ts
    b. lib.es5.d.ts 精读

  2. 一定要好好学一门后端语言

  3. 强化自己使用
    千万不要使用cli,tsconfig一定要自己配
    react中使用TypeScript比较难搞，会多一些Dom

  4. 学习ts的四个阶段
    ● ts基础夯实
    ● node开发 相对标准
    ● 原生dom开发
    ● sdk阶段

TypeScript实现接口

/**
 * 1. interface和type的区别
 * 相同点
 * 1-1 都可以描述一个对象或函数
 * 1-2 都允许进行扩展
 * 不同点
 * 2-1 type 声明基本类型别名
 * 2-2 typeof获取实例的对象
 * 2-3 interface可以被合并
 */

/**
 * 1. 有关后台的接口，愿意去使用Interface
 * 2. 第三方和开发的SDK，比如vue
 * 3. 前端的库
 * 4. 正常的开发任务， type直接
 */
interface IPriceData {
    id: number;
    m:string;
}

type IPriceDataArray =IPriceData[];

function getPriceData() {
    return new Promise<IPriceDataArray>((resolve, reject) => {
        fetch('url').then(function(response) {
            return response.json();
        }).then(function(myJson){
            const data:IPriceDataArray = [];
            resolve(data)
        })
    })
}

TypeScript装饰器

js的IOC注入都是要靠它的，ts种要使用装饰器，要在tsconfig中通过"experimentalDecorators": true,来设置，然后要添加一个包reflect-metadata并且在tsconfig中通过"emitDecoratorMetadata": true打开。

import 'reflect-metadata';
function inject(serviceindex: string) {
    return function(target: Object,targetKey: string,index: number){
        console.log(target)
        Reflect.defineMetadata(serviceindex,'laoyuan666',target)
    }
}
class IndexController {
    public indexService: string;
    constructor(@inject('xxx') indexService: string) {
        this.indexService = indexService;
    }
}

const indexController = new IndexController('京程一灯')
console.log(indexController.indexService)
console.log(Reflect.getMetadata('xxx',IndexController))

高级类型

Extract => 选择给可分配给另一种类型的子类型
InstanceType => 获取构造函数的实例类型
NonNullable => 从类型种排除null和undefined
Parameters => 函数参数类型的元组
Partial => 将对象的所有属性设为可选
Required =>将对象所有属性设为必选
Readonly => 使对象种的所有属性为只读
Pick => 从一个复合类型中，取出几个想要的类型组合
Omit => 从一个类型中排除掉一些字段
Exclude => 从另一个类型中排除了一个类型(排除掉两个类型交集)
Record => 从键类型到值类型的映射

进阶

使用Webpack构建开发环境

  1. 安装webpack webpack-cli
  2. 加一个script: "client": "webpack --mode development"
  3. 安装swc-loader，babel-loader配置起来较麻烦
  4. script命令webpack –mode development
  5. 如果要考虑eslint可以使用eslint-config-airbnb-typescript
    // tsconfig.json
    {
        "compilerOptions": {
            "lib":["DOM","ES6"],
            "target": "ES6",
            "types":["reflect-metadata"],
            "noImplicitAny": true,
            "strict": true,
            "noImplicitThis": true,
            "experimentalDecorators": true,
            "emitDecoratorMetadata": true,
            "strictNullChecks": true
        }
    }
//webpack.config.js
const { resolve } = require('path')
module.exports = {
    entry: {
        main: resolve("./src/index.ts"),
    },
    module: {
        rules: [
            {
                test: /\.ts$/,
                exclude: /(node_modules|bower_components)/,
                use: {
                    // Use `.swcrc` to configure swc
                    loader: "swc-loader"
                }
            }
        ]
    },
}

使用经验

  1. vue2坚决不能用ts
  2. node这一块跟Java Php一样
  3. d.ts是声明文件，引入文件时如果没有声明文件时会出现三个小点，可以搜索@types/packageName找一下声明文件
  4. 除了babel以外的编译器，swc snowpack esbuild vite gulp
  5. vue3 ts vite开发确实快 esbuild esmodule
  6. webpack生态完善 v8-compile-cache+Sparkplug+commonJs+插件代码
  7. 开发 sdk 用rollup+parcel/snowpack + rome(ts合集，不好用)
